Primeiros Passos com HTML

O que é HTML?

HTML (Linguagem de Marcação de HiperTexto) é o bloco de construção mais básico da web. Define o significado e a estrutura do conteúdo da web. Outras tecnologias além do HTML geralmente são usadas para descrever a aparência/apresentação (CSS) ou a funcionalidade/ comportamento (JavaScript) de uma página da web. "Hipertexto" refere-se aos links que conectam páginas da Web entre si, seja dentro de um único site ou entre sites. Links são um aspecto fundamental da web. Ao carregar conteúdo na Internet e vinculá-lo a páginas criadas por outras pessoas, você se torna um participante ativo na world wide web. O HTML usa >strong<"Marcação">/strong< para anotar texto, imagem e outros conteúdos para exibição em um navegador da Web. A marcação HTML inclui "elementos" especiais, como >head<, >title<, >body<, >header<, >footer<, >article<, >section<, >p<, >div<, >span<, >img<, >aside<, >audio<, >canvas<, >datalist<, >details<, >embed<, >nav<, >output<, >progress<, >video<, >ul<, >ol<, >li< e muitos outros.

Trabalhando com formularios em HTML

Em HTML, formulários são usados para coletar dados do usuário. Eles são criados usando a tag >form< e contêm elementos como campos de entrada (textos, números, etc.), botões de envio, áreas de texto, menus dropdown, entre outros. A tag >form< define o início e o fim do formulário, e os atributos action e method especificam onde os dados serão enviados e como. Elementos de um formulário HTML: >form<: Define o início e o fim do formulário. >input<: Define campos de entrada, como texto, número, senha, e-mail, etc. >textarea<: Define áreas de texto maiores. >select<: e <option<: Define menus dropdown. >button<: Define botões. >label<: Define rótulos para os campos, melhorando a acessibilidade. >fieldset<: Agrupa elementos relacionados do formulário. >legend<: Define o título de um >fieldset<.

Estruturando seu HTML + Formatações

A estrutura básica de um documento HTML consiste em tags que definem o conteúdo e a organização da página web, incluindo o cabeçalho (HEAD) e o corpo (BODY). Além disso, a formatação de texto, como parágrafos e títulos, é feita com tags específicas para estruturar e apresentar o conteúdo de forma clara. Estrutura do Documento HTML: <!DOCTYPE html>: Define a versão do HTML utilizada. < html > : Elemento raiz que contém todo o conteúdo da página. < head > : Contém informações sobre o documento, como título, metadados e links para arquivos externos (CSS, JavaScript). < title > : Define o título da página, que aparece na aba do navegador. < body > : Contém o conteúdo visível da página, como texto, imagens e outros elementos. Formatação de Texto: < h1 > a <h6>: Define títulos e subtítulos, com <h1> sendo o nível mais alto. < p >: Define parágrafos de texto. <b>: Define texto em negrito. <i > : Define texto em itálico. <em > : Define texto enfatizado (geralmente itálico). <strong >: Define texto forte (geralmente negrito). Outras Formatações: < ul > , < ol > , < li >: Cria listas não ordenadas e ordenadas. <a >: Cria links para outras páginas ou recursos. <img>: Inserção de imagens. <table>, <tr>, < td >: Cria tabelas. <div>: Elemento genérico para agrupar conteúdo e aplicar estilos.

Trabalhando com midias usando HTML

Em HTML, a manipulação de mídias (imagens, áudio e vídeo) é feita através de tags específicas que permitem a incorporação e exibição desses elementos na página web. Imagens: A tag <img> é usada para inserir imagens. É recomendável especificar a largura e altura da imagem para evitar problemas de layout durante o carregamento, diz um artigo da DIO. A tag <img> deve conter o atributo src que aponta para o arquivo de imagem. Imagens SVG não perdem resolução ao serem redimensionadas, segundo um artigo da DIO. Áudio: A tag <audio> é usada para incorporar arquivos de áudio. Ela pode conter múltiplas fontes de áudio através da tag <source>, permitindo que o navegador escolha a mais compatível, segundo um artigo da DIO. Formatos de áudio suportados incluem MP3, WAV e OGG, de acordo com um artigo da DIO. O texto entre as tags <audio> e </audio> é exibido apenas em navegadores que não suportam a tag, conforme um artigo da DIO.

Criando tabelas com o HTML

Para criar tabelas em HTML, utiliza-se a tag <table> para definir a tabela, e dentro dela, as tags <tr> para criar linhas e <td> para criar células de dados. Se quiser cabeçalhos, usa-se a tag . Explicação: <table></table>: Define o início e o fim da tabela. <tr></tr>: Define uma linha na tabela. <th></th>: Define uma célula de cabeçalho (geralmente usada para títulos de colunas). <td></td>: Define uma célula de dados dentro de uma linha. Recursos adicionais: <caption></caption>: Adiciona uma legenda à tabela. <thead></thead>: Define a área do cabeçalho da tabela. <tbody></tbody>: Define a área do corpo da tabela. <tfoot></tfoot>: Define a área do rodapé da tabela. colspan: Mescla células horizontalmente. rowspan: Mescla células verticalmente. Para personalizar a aparência da tabela, pode-se usar CSS para adicionar cores, bordas, espaçamento, etc.

Entendendo HTML semântico

HTML semântico é a prática de usar elementos HTML que carregam um significado específico sobre o conteúdo que eles delimitam, em vez de apenas definir sua aparência. Em vez de depender apenas de divs e spans para estruturar uma página, o HTML semântico utiliza tags como <header>, <nav>, <article>, <section>, <footer>, etc., que descrevem o papel de cada seção do conteúdo. Isso melhora a acessibilidade, a usabilidade e o SEO de um site. O que é HTML semântico? Em essência, o HTML semântico é sobre dar significado ao código HTML. Em vez de usar elementos genéricos para estruturar um site, o HTML semântico utiliza elementos com significado intrínseco, como:

< header>
: Representa o cabeçalho do site ou de uma seção.
<nav>
: Define a seção de navegação principal do site.
<main>
: Envolve o conteúdo principal da página.
<article>
: Define um conteúdo autônomo e autoexplicativo, como um artigo ou post.
<section>
: Divide o conteúdo em seções temáticas.
<aside>
: Define conteúdo relacionado que pode ser considerado separado do conteúdo principal, como barras laterais ou anúncios.
<footer>
: Representa o rodapé do site ou de uma seção. Por que usar HTML semântico?

Acessibilidade:

As tags semânticas ajudam os leitores de tela e outras tecnologias assistivas a entender a estrutura da página, tornando o conteúdo mais acessível para pessoas com deficiência. SEO: Os motores de busca usam o HTML semântico para entender o conteúdo de uma página e indexá-la de forma mais eficaz.

Usabilidade:

A estrutura clara e organizada do HTML semântico facilita a navegação e a compreensão do conteúdo tanto para usuários quanto para desenvolvedores. Manutenção: Um código HTML semântico é mais fácil de entender e manter, o que pode economizar tempo e esforço no futuro.

Legibilidade:

A utilização de tags semânticas torna o código mais limpo e legível, facilitando a colaboração entre desenvolvedores.
